# Contributing

Thank you for your interest in contributing to ChameleonIQ!

## Reporting Issues

We use GitHub Issues for tracking bugs and feature requests:

-   **Feature requests**: https://github.com/EdAlita/ChameleonIQ/issues
-   **Bug reports**: https://github.com/EdAlita/ChameleonIQ/issues

Please include:
- Clear reproduction steps
- Expected vs actual
behavior
- Environment details (OS, version, configuration)
- Relevant logs or screenshots

## Development Setup

1.  Fork the repository
2.  Clone your fork:
   
    ```bash
    git clone https://github.com/your-username/ChameleonIQ.git
    ```
    
4.  Create a new branch:
   
    ```bash
    git checkout -b feature/your-feature-name
    ```

## Pull Request Process

-   Ensure your code follows project style guidelines
-   Add or update tests if necessary
-   Run pre-commit with all the hooks in the project
-   Make sure all tests pass
-   Provide a clear description of your changes
-   Link related issues in your PR

## Code Style

Follow existing code conventions and formatting.

## Questions?

Open an issue for discussion.
